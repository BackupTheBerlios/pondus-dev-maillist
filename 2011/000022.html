Hi,

nice program, thanks for the work!
I have a feature request though, as I just bought a scale that measures the percentage of fat, water and muscles. So it would be great to enter and plot those data, too, which should be no problem to integrate into the current data structure (all hail xml ;)).
If you're short on time, implementing multiple users (as requested in June 2010: https://lists.berlios.de/pipermail/pondus-dev/2010/000012.html) would allow for a workaround by adding three "users" and using each one for another value. You could even add "percentage" as an available dimension (besides metric and that deprecated system the americans use) and save that as a userspecific preference in the .xml file, just to have the proper dimensions on screen ^^

Well, until then, there's another workaround without the need to change anything, though you can't plot all four values side by side: Copy the user_data.xml file to, say, sven_gewicht.xml, sven_fett.xml and so on and call a script that takes one of these xml-files, renames it to user_data.xml, starts pondus and after pondus terminating re-renames it back to the original name. This is, of course, a workaround for the user feature as well, so if I should come to write that script, I'll post it here.

Either way, keep up the good work!
